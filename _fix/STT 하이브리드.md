# STT 품질 고도화: 하이브리드 전략 (Hybrid STT Strategy)

사용자 요청에 따라 실시간성은 유지하면서 정확도를 극대화하는 **'하이브리드 전사 시스템'**으로 업그레이드합니다. 구현 전 현재 상태와 변경될 구조를 정리했습니다.

## 1. 현재 시스템 현황 (As-Is)
- **핵심 엔진**: `Faster-Whisper` (Large-v3-turbo) 단독 실행
- **작동 방식**:
  1. 마이크로 입력된 음성을 `Faster-Whisper`에 전달
  2. 들리는 소리 그대로 텍스트로 변환하여 즉시 화면에 표시
- **한계점 (현재 겪고 계신 문제)**:
  - 앞뒤 문맥을 파악하지 못해 발음이 비슷한 엉뚱한 단어로 인식됨
  - 전문 용어(IT 기술 용어 등)가 일반 명사로 잘못 기록됨
  - **결과**: "말은 빠른데 내용은 부정확함"

---

## 2. 하이브리드 전략 도입 (To-Be)
속도를 위한 **STT**와 정확도를 위한 **LLM**을 분리하여 **"일단 보여주고, 뒤에서 고치는"** 2단계 파이프라인을 구축합니다.

### 🔄 1단계: 실시간 전사 (Fast & Rough)
- **역할**: 사용자가 말을 하는 즉시 반응
- **엔진**: Faster-Whisper (기존 유지)
- **변경 없음**: 0.5초 이내에 화면에 텍스트가 뜨는 경험은 그대로 유지됩니다.

### ✨ 2단계: 비동기 LLM 문맥 보정 (Slow & Accurate)
- **역할**: 전사된 텍스트를 문맥에 맞게 "성형 수술"
- **엔진**: Gemma 2 (Local LLM)
- **작동 방식**:
  1. STT가 문장을 내뱉으면, 백그라운드에서 조용히 LLM에게 보냄
  2. LLM에게 **"이 문장은 회의 내용이야. 문맥상 어색한 단어나 오타를 고쳐줘"**라고 명령
  3. LLM이 고친 문장이 도착하면, 화면에 떠 있던 기존 텍스트를 **새로운 텍스트로 교체**
- **기대 효과**:
  - "서버바 죽었다" (STT) ➡️ **"서버가 죽었다"** (LLM 교정)
  - "기터브에 푸시해" (STT) ➡️ **"GitHub에 Push해"** (LLM 교정)

## 3. 구현 로드맵
이 계획이 승인되면 다음 순서로 코드를 수정합니다.

1.  **[Backend] LLM 교정 서비스 개발**: 
    - 단순 요약이 아닌, '문장 교정 전용' 프롬프트 개발 ("사실을 왜곡하지 말고 오타만 고쳐라")
2.  **[Backend] 파이프라인 연결**: 
    - STT 완료 시점에 LLM 교정 태스크를 비동기(Fire-and-forget)로 실행
3.  **[Frontend] UI 업데이트 로직**: 
    - 처음에 뜬 텍스트가 나중에 수정된 텍스트로 자연스럽게 바뀌도록 소켓 처리 로직 개선
