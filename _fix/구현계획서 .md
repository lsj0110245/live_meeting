# 구현 계획서: 자동 회의록 고도화 및 메타데이터 동기화

회의가 종료된 직후(웹소켓 연결 종료 시), 사용자가 **버튼을 누를 필요 없이 AI가 자동으로** 회의 내용을 분석하여 전문적인 회의록을 생성하고 시스템 정보를 업데이트하는 프로세스를 구현합니다.

## 제안된 변경 사항

### 1. 백엔드: LLM 프롬프트 및 분석 고도화
- **정형 서식 강제**: AI가 응답할 때 반드시 사용자님이 제시한 마크다운 구조를 따르도록 프롬프트를 수정합니다.
  - `# {제목} 회의록`
  - `## 📅 요약`
  - `## 📌 주요 안건 및 내용` (리스트 대신 `-` 불렛 포인트 사용)
  - `## ✅ 결론 및 결정 사항`
  - `## 📝 향후 계획`
- **리스트 출력 문제 해결**: `['내용']`과 같은 기술적인 리스트 형식이 절대 노출되지 않도록 자연스러운 문장 및 불렛 형식을 강제합니다.
- **메타데이터 추출**: 요약 내용과 별개로 제목, 회의 유형, 참석자 명단을 JSON 형태로 함께 추출합니다.

### 2. 백엔드: 메타데이터 자동 동기화 로직
- **자동 업데이트**: [final_cleanup_and_summary](file:///c:/big20/live_meeting/backend/app/api/endpoints/recording.py#377-425) 백그라운드 태스크에서 AI 분석 결과가 나오면 [meetings](file:///c:/big20/live_meeting/backend/app/api/endpoints/meeting.py#23-42) 테이블의 [title](file:///c:/big20/live_meeting/backend/app/utils.py#4-21), `attendees`, `meeting_type` 필드를 즉시 업데이트합니다.
- **데이터 저장**: 최종 생성된 마크다운 문자열을 [Summary](file:///c:/big20/live_meeting/backend/app/models/summary.py#6-22) 모델에 저장하여 상세 페이지에서 바로 보여줄 수 있게 합니다.

### 3. 프런트엔드: 시각적 개선
- **Markdown 렌더링**: 상세 페이지의 요약 섹션에서 마크다운 문법(헤더, 불렛 등)이 깨지지 않고 예쁘게 보이도록 스타일을 점검합니다.
- **상태 알림**: 회의록 생성이 진행되는 동안 유저에게 "AI가 회의록을 정리하고 있습니다..."라는 메시지를 명확히 보여줍니다.

## 검증 계획

### 수동 검증
1. **실시간 테스트**: 2분 내외의 회의를 진행하고 종료합니다.
2. **자동 동기화 확인**: 회의 목록으로 나갔을 때 제목이 전문적인 제목으로 바뀌어 있는지 확인합니다.
3. **요약 포맷 확인**: 상세 페이지에서 `📅`, `📌` 등 이모지와 함께 불렛 포인트로 정리된 회의록이 나오는지 확인합니다. (리스트 괄호 `[]`가 없는지 확인)
