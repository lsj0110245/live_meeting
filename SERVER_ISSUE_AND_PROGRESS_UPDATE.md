# 서버 연결 이슈 및 분석 진행률 기능 수정 내역 보고서

## 1. 서버 연결 끊김 및 데이터 처리 이슈

### **1.1 문제 상황**
- 녹음 중 붉은색 "연결 끊김" 상태가 되거나, 녹음 종료(`stopRecording`) 시 데이터가 제대로 저장되지 않는 현상이 발생.
- 실시간 녹음 화면의 "버퍼링 바"가 0%에서 움직이지 않는 현상 발생.

### **1.2 원인 분석**
1. **WebSocket 메시지 타입 누락**: 
   - 백엔드에서 프론트엔드로 버퍼링 상태(`buffer_seconds`)를 보낼 때, 메시지 타입(`type: "buffering"`)을 포함하지 않아 프론트엔드가 이를 무시함. 이로 인해 버퍼링 바가 갱신되지 않음.
2. **데이터 전송 포맷 불명확**:
   - `MediaRecorder`의 `Blob` 데이터를 `websocket.send()`로 보낼 때, 명시적인 바이너리 변환 없이 전송하여 일부 환경에서 데이터 처리가 불안정할 가능성이 있었음.
3. **종료 시그널 처리 미흡**:
   - 녹음 종료 시 `websocket.close()`가 즉시 호출되어, 서버가 마지막 데이터를 처리할 시간을 확보하지 못하고 연결이 끊기는 경우가 발생할 수 있음.

### **1.3 수정 내역**
- **녹음 중 버퍼링 바 수정 (`recording.py`, `recording.js`)**:
  - **백엔드**: WebSocket 메시지 전송 시 `{"type": "buffering", ...}` 필드를 명시적으로 추가.
  - **프론트엔드**: `buffering` 메시지를 수신했을 때 정확히 UI를 업데이트하도록 로직 검증.
- **데이터 전송 안정화 (`recording.js`)**:
  - `websocket.send(event.data)` 대신 `event.data.arrayBuffer()`를 사용하여 **명시적인 ArrayBuffer 바이너리 전송**으로 변경.
- **종료 로직 개선 (`recording.js`)**:
  - `stopRecording` 함수 내에서 소켓을 닫기 전, `{"type": "stop_recording"}` 메시지를 서버에 먼저 전송하여 서버가 남은 작업을 마무리(`final_cleanup`)할 수 있도록 신호를 보냄.

## 2. 분석 진행률 (AI Processing Progress) 구현

### **2.1 문제 상황**
- 녹음 종료 후 대시보드 목록에서 "AI 분석 중" 상태일 때, 진행률이 계속 **0%**로 유지되다가 작업 완료 시 갑자기 **100%**로 점프함.
- 사용자는 작업이 진행 중인지 멈춘 것인지 알 수 없어 답답함을 느낌.

### **2.2 원인 분석**
- **백엔드 진행률 업데이트 누락**: 
  - `progress_service`는 구현되어 있었으나, 실제 무거운 작업을 수행하는 **Worker(`meeting_tasks.py`)** 에서 진행률을 업데이트하는 코드가 **전혀 없었음**.
  - 따라서 프론트엔드가 1초마다 진행률을 요청해도 항상 초기값(0)만 반환됨.

### **2.3 수정 내역**
- **백엔드 작업 단계별 진행률 업데이트 추가 (`meeting_tasks.py`)**:
  `process_meeting_summary` 함수 내 주요 작업 단계마다 `progress_service.set_progress(mid, percent)` 호출을 추가하여 실시간 진행 상황을 반영.

| 진행률 (%) | 작업 단계 | 설명 |
| :---: | :--- | :--- |
| **10%** | **작업 시작** | 백그라운드 태스크 진입 및 초기화 |
| **20%** | **데이터 로드** | DB에서 전사 데이터(Transcript) 조회 완료 |
| **30%** | **요약 시작** | LLM(AI)에 요약 생성을 요청하는 단계 |
| **30~90%** | **요약 생성 중** | **[개선]** AI 처리 중 Bar가 멈추지 않도록 **약 2초마다 1%씩 서서히 증가** (심리적 안정감 제공) |
| **90%** | **요약 완료** | LLM 응답 수신 완료 및 파싱 |
| **95%** | **메타데이터** | 회의 제목, 참석자 등 메타데이터 추출 및 업데이트 |
| **100%** | **저장 완료** | 최종 요약본 DB 저장 완료 (분석 종료) |

### **추가 UX 개선 사항**
- **100% 시각적 피드백**: 프론트엔드에서 100% 도달 시 즉시 새로고침하지 않고, **0.5초 동안 완료 상태(100%)를 유지**하여 사용자에게 완료되었음을 명확히 인지시킨 후 목록을 갱신합니다.
- **실시간 바 애니메이션**: 30%에서 90% 사이의 부드러운 상승 로직을 통해 서비스가 중단되지 않고 정상 작동 중임을 직관적으로 전달합니다.

이제 대시보드에서 분석 중인 회의를 볼 때, 위 단계에 따라 진행률 바가 0%에서 100%까지 단계적으로 차오르는 것을 확인할 수 있습니다.
