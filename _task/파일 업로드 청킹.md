### 2. 텍스트 청킹 (요약 단계)

#### [MODIFY] [meeting_tasks.py](file:///c:/big20/live_meeting/backend/app/services/meeting_tasks.py)

**Map-Reduce 전략:**

**Step 1: Map (부분 요약)**
- 전사 텍스트를 3,000자 청크로 분할 (500자 오버랩)
- 각 청크를 LLM에 보내 부분 요약 생성
- 결과: `["요약1", "요약2", "요약3", ...]`

**Step 2: Reduce (통합 요약)**
- 모든 부분 요약을 하나로 합침
- 다시 LLM에 보내 최종 통합 요약 생성
- 결과: 전체 회의 요약 (JSON)

**구현 세부사항:**
```python
def chunk_text(text, chunk_size=3000, overlap=500):
    # 텍스트를 오버랩하며 분할
    chunks = []
    start = 0
    while start < len(text):
        end = start + chunk_size
        chunks.append(text[start:end])
        start += (chunk_size - overlap)
    return chunks

# 1. 청크 분할
chunks = chunk_text(full_text)

# 2. 각 청크 요약
partial_summaries = []
for chunk in chunks:
    summary = await llm_service.generate_simple_summary(chunk)
    partial_summaries.append(summary)

# 3. 통합 요약
combined = "\n\n".join(partial_summaries)
final_summary = await llm_service.generate_summary(title, combined)
```

## 검증 계획

### 테스트 시나리오
1. **짧은 회의 (5분)**: 기존 방식과 동일하게 작동 확인
2. **긴 회의 (30분+)**: 전체 내용 반영 확인
3. **매우 긴 회의 (1시간+)**: 성능 및 품질 확인

### 성공 기준
- ✅ 전체 회의 내용이 요약에 반영됨
- ✅ 처리 시간이 합리적 범위 내 (회의 길이의 2배 이내)
- ✅ 오버랩으로 인한 중복 내용 최소화
